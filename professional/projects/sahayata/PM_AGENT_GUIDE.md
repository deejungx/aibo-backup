# How to Use Your Sub-Agents

## Sahayata Product Manager (Persistent)

Your PM agent is **always available** for product strategy, backlog organization, and alignment.

### How to Access

**Quick Command (next conversation):**
```
"Talk to my PM agent about [topic]"
or
"PM agent: [question]"
or just ask me and I'll route to the PM agent if relevant
```

**What to Ask:**
- Backlog prioritization ("What should we build first?")
- Feature dependencies ("Does X block Y?")
- Cross-org conflicts ("How do we align Symbiosis + Telerivet on this?")
- Blocker resolution ("How do we unblock IP ownership?")
- Customer discovery synthesis ("What patterns are we seeing from Arun's discovery?")
- Roadmap strategy ("What's the 12-week plan given our constraints?")

### What the PM Agent Knows

- Full team structure (Symbiosis + Telerivet)
- Partnership model + IP issues
- Arun's customer discovery work
- Technical status (polling just shipped)
- Business constraints + market context
- All previous strategic decisions

### What You Get Back

- Ranked backlog (clear priorities)
- Feature dependency map
- Milestone roadmap
- Blocker resolution plans
- Cross-org alignment recommendations
- Customer-driven insights

---

## When to Use the PM Agent vs. Ask Me Directly

**Use PM Agent when:**
- Clarifying product priorities
- Organizing large backlogs
- Making strategic calls with trade-offs
- Synthesizing cross-org feedback
- Planning 3+ month roadmap

**Ask me directly when:**
- Need quick facts/status
- Checking code or repos
- Writing/organizing docs
- Need to remember past conversations
- Want daily standup/heartbeat

---

The PM agent has your back for the hard product thinking. Use it liberally.
